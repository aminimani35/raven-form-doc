# RavenRepeater

`RavenRepeater` is the component that powers `type: "repeater"` fields inside `RavenForm` and `RavenWizard`. It is also exported as a standalone component for advanced compositions.

## Import

```ts
import { RavenRepeaterField } from "raven-form";
```

## Props

```ts
interface RavenRepeaterProps {
  field: FormField; // FormField with type="repeater" and a repeaterConfig
  adapter: FormAdapter;
  ui: UIAdapter;
}
```

## Example (standalone)

```tsx
const contactsField: FormField = {
  name: 'contacts',
  type: 'repeater',
  label: 'Contacts',
  repeaterConfig: {
    minRows: 1,
    maxRows: 10,
    addLabel: '+ Add Contact',
    fields: [
      { name: 'name',  type: 'text', label: 'Name',  colSpan: 6 },
      { name: 'email', type: 'email',label: 'Email', colSpan: 6 },
    ],
  },
}

<RavenRepeaterField
  field={contactsField}
  adapter={RHFAdapter}
  ui={ShadCNUIAdapter}
/>
```

## Internals

```
RavenRepeater
  ├── Header: field label + row count badge
  ├── for each row (rowId):
  │     RepeaterRow
  │       ├── Row number badge  (#1, #2, ...)
  │       ├── 12-col sub-grid
  │       │     for each sub-field:
  │       │       InlineSmartField
  │       │         └── ui.FormItem + ui.Input / ui.Select / ...
  │       └── Remove button (hidden when rows ≤ minRows)
  └── Add Row button (hidden when rows ≥ maxRows)
```

## Row ID management

Each row is assigned a stable internal ID (`row_1`, `row_2`, …) generated by a monotonically increasing counter. Removing a row removes its ID from the internal list — Raven Form uses the ID as the React `key` to correctly unmount removed rows.

## Field name scoping

Sub-field names are automatically prefixed with the parent field name and row index:

```
contacts[0].name
contacts[0].email
contacts[1].name
contacts[1].email
```

This is compatible with React Hook Form's dot-bracket path notation.
